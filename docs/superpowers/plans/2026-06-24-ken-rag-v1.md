# ken-rag v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a local, private, terminal-native RAG CLI (`ken`) that indexes docs + codebases and answers questions with citations, 100% offline, aimed at 5k+ GitHub stars via the code-aware-chunking moat, git-awareness, and a Claude-Code-quality TUI.

**Architecture:** Hand-written ~200-line pipeline (no LangChain). Protocol-based dependency injection: pure frozen `domain/` models + protocols, concrete impls depend only on protocols, assembled in one DI root (`cli/context.py`). Repository pattern over LanceDB. Two Ollama models — nomic-embed-text (768-dim, retrieval) and qwen2.5 (generation). Hybrid retrieval (dense + LanceDB FTS, RRF-fused) from v1.

**Tech Stack:** Python 3.10+, Ollama, nomic-embed-text, qwen2.5:{3b,1.5b}/llama3.2:1b, LanceDB, Typer, Textual+Rich, PyMuPDF, tree-sitter-language-pack, pathspec, psutil, httpx. Tests: pytest + respx. Packaging: uv/pipx, publish via GitHub Actions.

**Source of truth:** `ken-rag-spec.md` (goal/scope) + `docs/superpowers/specs/2026-06-24-ken-rag-architecture-draft.md` (architecture + eng-review decisions).

**Build mode:** Phase-by-phase with a STOP checkpoint after each phase for user review.

---

## Eng-review invariants (apply to every relevant task)
1. **nomic prefixes:** documents embedded as `search_document: <text>`, queries as `search_query: <text>`. Never embed raw. nomic via Ollama does not auto-prefix.
2. **tree-sitter-language-pack** (not the dead `tree-sitter-languages`).
3. **Hybrid retrieval in v1:** DenseStage + KeywordStage (LanceDB FTS) fused with RRF.
4. **num_ctx ≥ 8192** threaded into every generation call. No code path bypasses it.
5. **Embedder guard:** store embedder name+dim in `ken_meta` at first ingest; re-validate on every ingest and query. Mismatch → typed error with fix-it message.
6. **Chunk lifecycle:** delete-by-file then upsert. Never in-place. PK = `file_path::chunk_index`. No orphans, no drift.
7. **Embed batching:** `embed_texts` batches ≤64 per HTTP call.
8. **Graceful degradation:** Ollama down / git absent / grammar missing → typed errors + fallbacks, never a traceback.

```python
class OllamaEmbedder:
    def __init__(self, client, model_name: str, dim: int = 768, batch_size: int = 64):
        self._client = client; self._model = model_name
        self._dim = dim; self._batch = batch_size

    @property
    def model_name(self) -> str: return self._model
    @property
    def dim(self) -> int: return self._dim

    def embed_texts(self, texts):
        out = []
        for i in range(0, len(texts), self._batch):
            batch = [f"search_document: {t}" for t in texts[i:i+self._batch]]
            vecs = self._client.embed(self._model, batch)
            out.extend(tuple(v) for v in self._validate(vecs))
        return out

    def embed_query(self, text):
        vecs = self._client.embed(self._model, [f"search_query: {text}"])
        return tuple(self._validate(vecs)[0])

    def _validate(self, vecs):
        from ken_rag.domain.errors import DimensionMismatchError
        for v in vecs:
            if len(v) != self._dim:
                raise DimensionMismatchError(
                    f"Embedder returned {len(v)}-dim, expected {self._dim}",
                    hint="Wrong embedding model? Expected nomic-embed-text (768-dim).")
        return vecs
```

```python
def rrf_fuse(rankings, k_const=60, top_k=5):
    scores, by_id = {}, {}
    for ranking in rankings:
        for rank, rc in enumerate(ranking):
            cid = f"{rc.chunk.file_path}::{rc.chunk.chunk_index}"
            scores[cid] = scores.get(cid, 0.0) + 1.0 / (k_const + rank)
            by_id[cid] = rc.chunk
    ordered = sorted(scores.items(), key=lambda kv: kv[1], reverse=True)[:top_k]
    return [RetrievedChunk(chunk=by_id[cid], score=s) for cid, s in ordered]
```

## TUI / CLI Design Spec (design review, locked 2026-06-24)

> Applies to Phase 5 (`chat`) and the Rich output of `ask`/`search`/`list`/`status`. Terminal UI — specs are ASCII-layout, not web. Implement the shared theme in `cli/render.py` and reuse everywhere.

### Theme (define once in `render.py`, reuse in every command)
- **Signature accent: cyan/teal** — prompts (`›`, `▸`), citation numbers (`①②`), active selections. Nothing else uses it.
- `dim` for metadata (paths, line ranges, counts, key-hints). `red` ONLY for errors. Default fg for body text + answers.
- One frame max (box-drawing). No emoji-as-bullets, no multi-color. Restraint = the Claude-Code look.
- Respect `NO_COLOR` env (strip all color) and detect non-TTY (pipe) → plain text, no spinners/frames.
- Rich `Theme` object: `{"accent":"cyan", "meta":"dim", "error":"bold red", "user":"cyan", "ken":"default"}`.

### Citation format (shared `format_citation(chunk)` — used by chat, ask, search)
Inline numbered markers in answer text + a dim citation block under each answer:
```
① auth/middleware.py:14-37  require_auth
② auth/tokens.py:5-22       verify_token
```
Path is dim, line range dim, `symbol_name` accent. Numbers `①②③…` are accent. Never truncate the path — wrap if narrow. `--verbose` (ask/search) additionally prints the retrieved chunk text under each citation.

### `ken chat` — single-stream layout, input pinned
```
┌ ken ───────────────────────────── qwen2.5:3b · 142 chunks ┐
│  ▸ you                                                     │
│    how does auth middleware work?                          │
│                                                            │
│  ▸ ken                                                     │
│    The require_auth decorator wraps each route… ▌          │  streaming cursor ▌
│    ① auth/middleware.py:14-37  require_auth               │  inline citations
├────────────────────────────────────────────────────────── ┤
│ › ask about your code…                          ⏎ send     │  input pinned bottom
│ /sources  /clear  /exit            ↑ history   ⎋ quit       │  dim key-hints row
└──────────────────────────────────────────────────────────┘
```
- Single full-width conversation stream (NOT a split sources panel). `/sources` opens an **overlay** showing the last answer's full citations + chunk text, dismissed with `⎋`.
- Header shows active model + indexed chunk count (ambient trust).
- Key-hints row is always visible (keyboard discoverability — users don't read docs).

### Interaction states (every state designed, not just happy path)
| State | What user sees |
|---|---|
| First run / empty | Centered `ken` wordmark + "Ask anything about your indexed files." + 2 example questions + `142 chunks from 18 files indexed`. Never a blank pane. |
| Retrieving | Status line `⠋ searching 142 chunks…` (accent spinner), input disabled briefly |
| First-token wait | Spinner → `⠋ thinking…`, replaced instantly by first token |
| Streaming | Tokens append live, block cursor `▌`, markdown/code blocks render as they close (Rich) |
| Empty retrieval | `No relevant chunks found. Try rephrasing, or run `ken add` to index more files.` |
| Ollama drops mid-stream | Keep partial answer + `⚠ Ollama stopped responding. Run `ollama serve`.` (red, no crash, history intact) |
| Model not pulled | `Model qwen2.5:3b not found — run `ollama pull qwen2.5:3b`` |

### `ken ask` one-shot output (the real first impression — most users hit this first)
```
The require_auth decorator wraps each route and checks the bearer
token before the handler runs, rejecting expired tokens with 401.

  ① auth/middleware.py:14-37  require_auth
  ② auth/tokens.py:5-22       verify_token
```
Answer block (default fg, markdown) → blank line → dim citation block. `--verbose` appends retrieved chunk text. Same `format_citation` as chat.

### Resize & keyboard a11y
- Textual reflows; spec floor: min 60×20. Below → drop frame + key-hint row, keep stream + input. Citations wrap (never truncate path).
- Keyboard: `↑/↓` input history · `PgUp/PgDn` scrollback · `Ctrl+C` cancel current stream (NOT quit) · `⎋` or `/exit` quit · `Ctrl+L` clear · `Enter` send · `Shift+Enter` newline (multi-line). No mouse required.

## Failure modes (each has test + handling + visible error)
| Codepath | Failure | Handled | User sees |
|---|---|---|---|

| Embedder | wrong dim/model | DimensionMismatchError | "Expected nomic 768-dim" |

| code_chunker | bad/unknown grammar | fallback_chunker | (silent, degrades to line-window) |
| git layer | no git/.git | NullGitClient | (silent, filesystem mode) |
